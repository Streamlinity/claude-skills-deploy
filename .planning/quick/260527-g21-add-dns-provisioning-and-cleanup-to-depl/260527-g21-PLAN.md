<objective>
Add DNS A record provisioning and cleanup to the setup-coolify skill so a new deployment is end-to-end automated: operator runs `init.sh` → answers DNS prompts → `/setup-coolify` creates Coolify apps AND the DNS records that make Let's Encrypt succeed → `cleanup-deployment.sh` removes both.

Currently DNS is the most error-prone step in `docs/setup-guide.md` (manual A records before HTTPS), and the cleanup script leaves orphaned records. This closes both gaps with a Cloudflare-first implementation extensible to other providers via a `lib-dns-api.sh` library following the `lib-coolify-api.sh` pattern.

Purpose: Eliminate the only remaining manual step in the new-domain bootstrap, and make E2E test runs leave zero orphaned resources.
Output: New `lib-dns-api.sh` library + extensions to validate/provision/init/e2e/cleanup + docs updates. Cloudflare provider only in this pass; provider dispatch shim makes adding more straightforward.
</objective>

   - Header: `#!/usr/bin/env bash`, one-line purpose comment, `set -euo pipefail`, `: "${COOLIFY_REGISTRY:=$HOME/.claude/coolify.json}"`.
   - Functions to export (Cloudflare provider only; provider dispatch by `$DNS_PROVIDER` switch where it matters):
     - `dns_load_credentials <coolify_yaml_path>` — parses `coolify.yaml` `dns:` block (via inline python3 + pyyaml), reads the named credential from either `~/.claude/coolify.json` (if `credential_source: coolify_json`) OR Doppler (if `credential_source: doppler`, default). Exports `DNS_PROVIDER`, `DNS_ZONE_NAME`, `DNS_API_TOKEN`. Errors with named-field messages if anything is missing. For Doppler source, accepts `DOPPLER_PROJECT` + `DOPPLER_ENV` (use staging config — DNS credentials are shared across envs) and calls `doppler secrets get ... --plain` to fetch the token value.
     - `dns_check_credentials <coolify_yaml_path>` — same as above but only verifies non-empty token without exporting; returns 0/1. Used by validate.sh.
     - `dns_cf_curl <method> <path> [body]` — Cloudflare REST wrapper. Base URL `https://api.cloudflare.com/client/v4`. Header `Authorization: Bearer $DNS_API_TOKEN`. Same shape as `coolify_curl`.
     - `dns_cf_get_zone_id <zone_name>` — `GET /zones?name=<zone_name>` → first result `id`. Empty string if not found.
     - `dns_cf_find_record <zone_id> <name> <type>` — `GET /zones/<zone_id>/dns_records?name=<name>&type=<type>` → first result `id`. Empty if not found (idempotency lookup).
     - `dns_cf_upsert_a_record <zone_id> <name> <ip>` — find existing A record; if absent POST `/zones/<zone_id>/dns_records` with `{"type":"A","name":"<name>","content":"<ip>","ttl":300,"proxied":false}`; if present PATCH `/zones/<zone_id>/dns_records/<rec_id>` with `{"content":"<ip>"}`. Echoes the record_id to stdout (caller captures for report).
     - `dns_cf_delete_record <zone_id> <record_id>` — DELETE `/zones/<zone_id>/dns_records/<record_id>`. Tolerates 404 (already gone).
     - `dns_upsert_a_record <fqdn> <ip>` — provider-dispatching wrapper. Computes zone_name from `$DNS_ZONE_NAME` (must be a suffix of `$fqdn`; hard-fail otherwise with "ERROR: fqdn '<fqdn>' is not under configured DNS zone '<zone>'"). Calls `dns_cf_get_zone_id` then `dns_cf_upsert_a_record`. Echoes `<zone_id>|<record_id>` to stdout.
     - `dns_delete_record <zone_id> <record_id>` — dispatches to `dns_cf_delete_record` (only cloudflare in scope; switch on `$DNS_PROVIDER`).
   - Round-trip verification helper `dns_verify_a_record <fqdn> <expected_ip>` — `dns_cf_find_record` + GET record + confirm `content == expected_ip`. Hard-fail with stderr message if not — mirrors the volume mount round-trip pattern in provision.sh.
