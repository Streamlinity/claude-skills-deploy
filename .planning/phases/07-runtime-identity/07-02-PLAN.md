<!-- Existing Dockerfile.doppler.snippet structure — executor must prepend, not replace. -->

Current file opens with a comment header block:
```
# ============================================================================
# Doppler CLI install + ENTRYPOINT — paste into your Dockerfile.
# ...
# ============================================================================
```
Then contains `RUN apt-get update ...` (Doppler CLI install), `RUN mkdir -p /etc/doppler-cache`, and `ENTRYPOINT [...]`.

The ARG + LABEL stanzas must be placed BEFORE this existing content (ARGs must precede the RUN instruction that uses build context, and labels are typically near the top).

In a Dockerfile, ARG defined before FROM is only valid in FROM; ARG defined after FROM is available in RUN/LABEL/CMD. Since this snippet is pasted into the final/runtime stage (after the base FROM), the ARGs go at the top of the snippet — before the RUN block.

OCI label standard (opencontainers.org):
- `org.opencontainers.image.revision` — VCS commit SHA
- `org.opencontainers.image.created` — RFC 3339 timestamp of build
