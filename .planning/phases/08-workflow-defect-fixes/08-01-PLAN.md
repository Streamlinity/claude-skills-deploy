The file is a bash heredoc emitter. The `needs:` line lives inside the heredoc (between `cat > "$OUT_PATH" << YAML` and the closing `YAML`). It is YAML content emitted verbatim, not bash. No escaping required for this line.

The `TAG` variable is declared in the job's `env:` block two lines below the `needs:` line:
  TAG: \${{ needs.build.outputs.tag }}

Without `build` in `needs`, GitHub Actions ignores this reference and TAG becomes "".
