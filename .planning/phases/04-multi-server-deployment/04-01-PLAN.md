<objective>
Extend provision.sh, validate.sh, and lib-coolify-api.sh to support an optional
deploy_server: field in coolify.yaml and deploy_ssh_host/deploy_vps_ip fields in
coolify.json server entries. When set, Coolify apps are created on the named
server (not localhost), Docker volumes are created on the deployment VPS, and
DNS A records point to the deployment VPS public IP. When absent, full
backward compatibility — existing repos work unchanged.

Purpose: A coolify.yaml with `deploy_server: my-app-vps` deploys to a separate
VPS registered in Coolify; a coolify.yaml without it deploys to localhost
exactly as today. validate.sh fails loudly when deploy_server names a server
that does not exist.

Output: Three modified scripts implementing the DEPLOY_SERVER_NAME +
DEPLOY_SSH_HOST + DEPLOY_VPS_IP resolution chains. No new files, no new
dependencies.
</objective>
